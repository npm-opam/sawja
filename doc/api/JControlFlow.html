val resolve_class : 'a JProgram.program -> Javalib_pack.JBasics.class_name -> 'a JProgram.node
resolve_class p cn returns the class named cn in program p, if any.
Raises NoClassDefFoundError if p does not contain a class named cn.
val resolve_method : Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node -> 'a JProgram.node
resolve_method ms c returns the class or interface that defines the method ms, if any. If c does not define ms, then ms is searched recursivly in superclasses and interfaces of c. The caller is responsible to check that the class and the method defined in the class are visible from the current class.
Raises NoSuchMethodError if the method is not found
val resolve_method' : Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node -> 'a JProgram.class_node
resolve_method' ms c only look for the method in c and its superclasses.
val resolve_interface_method : Javalib_pack.JBasics.method_signature ->
       'a JProgram.interface_node -> 'a JProgram.node
resolve_interface_method ms i returns the interface that defines the method ms, or java.lang.Object if no interface defines ms but Object does. The caller is responsible to check that the interface and the method defined in the interface are visible from the current class.
Raises NoSuchMethodError if the method is not found.
val resolve_interface_method' : ?acc:'a JProgram.interface_node list ->
       Javalib_pack.JBasics.method_signature ->
       'a JProgram.node -> 'a JProgram.interface_node list
resolve_interface_method' ms c looks for the methods ms in c interfaces and recursively in their super-interfaces, stopping at every first occurence in each hierarchy. It returns the list of interfaces that defines ms.
val resolve_field : Javalib_pack.JBasics.field_signature ->
       'a JProgram.node -> 'a JProgram.node list
resolve_field fsi c returns the list of classes and interfaces that define the field (c,fsi), if any. See the JVM Spec for the algorithm. Several interfaces may define the same field and it is not specified which one to take (there is no order relation on direct interfaces), so this function returns a list.
See also Field Resolution
exception AmbiguousFieldResolution of Javalib_pack.JBasics.class_name list
Exception throwed when field cannot be exactly resolved. In this case a list of possible node is given.
val resolve_field_strong : Javalib_pack.JBasics.field_signature -> 'a JProgram.node -> 'a JProgram.node
resolve_field_strong fs node Return the class where the field fs used by class node is defined. If the resolution is ambigous, throw an AmbiguousFieldResolution exception. Use resolve_field internally.
val static_lookup_virtual : 'a JProgram.program ->
       Javalib_pack.JBasics.object_type ->
       Javalib_pack.JBasics.method_signature -> 'a JProgram.node list
static_lookup_virtual p obj ms returns the classes or interfaces that defines the method ms, if any. The caller is responsible to check that the class and the method defined in the class are visible from the current class.
Raises IncompatibleClassChangeError if the type obj is the type of an interface.
val static_lookup_interface : 'a JProgram.program ->
       Javalib_pack.JBasics.class_name ->
       Javalib_pack.JBasics.method_signature -> 'a JProgram.node list
static_lookup_interface p cs ms returns the classes or interfaces that defines the method ms, if any. The caller is responsible to check that the class and the method defined in the class are visible from the current class.
val static_lookup_special : 'a JProgram.program ->
       'a JProgram.node ->
       Javalib_pack.JBasics.class_name ->
       Javalib_pack.JBasics.method_signature -> 'a JProgram.class_node
static_lookup_special program c cs ms returns the defining class of the method that would be reached by executing "invokespecial cs.ms" with c as the current class.
val static_lookup_static : 'a JProgram.program ->
       Javalib_pack.JBasics.class_name ->
       Javalib_pack.JBasics.method_signature -> 'a JProgram.node
static_lookup_static p cs ms returns the classe that defines the method ms. The caller is responsible to check that the class and the method defined in the class are visible from the current class.

Utility functions using lookup and resolution


val implements_methods : Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node -> 'a JProgram.interface_node list
implements_methods ms c looks for the interfaces that defines method ms in the direct interfaces of c and recursively in their super-interfaces. If i1 and i2 defines ms and i1 extends i2, then i1 is before i2 in the result list.
Raises Not_found if ms cannot be found in c
val resolve_all_interface_methods : Javalib_pack.JBasics.method_signature ->
       'a JProgram.interface_node -> 'a JProgram.interface_node list
resolve_all_interface_methods ms i returns the list of interfaces that defines the method ms looking for ms in i and then recursivly in its super-interfaces. If i1 and i2 defines ms and i1 extends i2, then i1 is before i2 in the result list. The caller is responsible to check that the interface and the method defined in the interface are visible from the current class.
val overrides_methods : Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node -> 'a JProgram.class_node list
overrides_methods ms c looks for the classes that define methods that are overridden by (c,ms) (in the parents of c). The result list is ordered such that c1 is before c2 iff c1 extends c2.
Raises Not_found if ms cannot be found in c
val overridden_by_methods : Javalib_pack.JBasics.method_signature ->
       'a JProgram.node -> 'a JProgram.class_node list
overridden_by_methods ms c looks for the classes that define methods that overrides (or implements) (c,ms) (in the children of c).
Raises Invalid_argument("overridden_by_methods") if the method is a class or instance initialization method.
val implements_method : 'a JProgram.class_node -> Javalib_pack.JBasics.method_signature -> bool
implements_method c ms returns true iff the class has a method with the signature ms and which is not abstract. (Note: The method can be native.)
val implements_interface_or_subinterface : 'a JProgram.node -> 'a JProgram.interface_node -> bool
implements_interface_or_subinterface n i looks if n or one of its interfaces transitively implements i, it returns true if it is the case.
val implements_interface_or_subinterface_transitively : 'a JProgram.class_node -> 'a JProgram.interface_node -> bool
implements_interface_or_subinterface_transitively n i looks if n or one of its interfaces transitively implements i, if it is not the case it does it looks transitively on superclasses of n. It returns true if n transitively implements i.

Invoke lookup algorithms


val invoke_virtual_lookup : ?c:'a JProgram.class_node option ->
       Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node Javalib_pack.JBasics.ClassMap.t ->
       ('a JProgram.class_node * 'a Javalib_pack.Javalib.concrete_method)
       Javalib_pack.JBasics.ClassMethodMap.t
invoke_virtual_lookup ?c ms instantiated_classes returns the result of the InvokeVirtual instruction on a class c with the method signature ms, given a set of possibly instantiated classes.

Setting a value to c will only check (assert) that each class in instantiated_classes extends c. It raises an Assert_failure if the assertion fails and the program has been compiled without deactivating assertions.

val invoke_interface_lookup : ?i:'a JProgram.interface_node option ->
       Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node Javalib_pack.JBasics.ClassMap.t ->
       ('a JProgram.class_node * 'a Javalib_pack.Javalib.concrete_method)
       Javalib_pack.JBasics.ClassMethodMap.t
invoke_interface_lookup ?i ms instantiated_classes returns the result of the InvokeInterface instruction on an interface i with the method signature ms, given a set of possibly instantiated classes.

Setting a value to i will only check (assert) that each class in instantiated_classes implements i directly or indirectly. It raises an Assert_failure if the assertion fails and the program has been compiled without deactivating assertions.

val invoke_special_lookup : 'a JProgram.node ->
       'a JProgram.class_node ->
       Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node * 'a Javalib_pack.Javalib.concrete_method
invoke_special_lookup current_class c ms returns the result of the InvokeSpecial instruction on a class c with the method signature ms, from the class current_class.
val invoke_static_lookup : 'a JProgram.class_node ->
       Javalib_pack.JBasics.method_signature ->
       'a JProgram.class_node * 'a Javalib_pack.Javalib.concrete_method
invoke_static_lookup c ms returns the result of the InvokeStatic instruction on a class c with the method signature ms.
val static_lookup' : 'a JProgram.program -> 'a PP.t -> 'a PP.t list
static_lookup' program pp returns a list of methods that may be called from program point pp (provided by one of the PP_* modules). The computation is based on RTA or CHA, depending on the function used to build the program (it uses the field program.static_lookup_method). Caution: If the program code representation has been changed, program.static_lookup_method must have been updated at the same time (see JProgram.map_program function).
